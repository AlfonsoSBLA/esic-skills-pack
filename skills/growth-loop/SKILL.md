# /growth-loop — Conversacional

## Pattern

1. **Acoge** — confirma negocio
2. **Diagnose** — 4 preguntas
3. **Confirma** — espejo
4. **Produce** — loop + sub-loops + scale lever + guardrail
5. **Itera** — ¿es real loop o funnel disfrazado?

## Flujo

### Q1 · Core action
*"¿Acción que más valor crea para el usuario? (publicar · invitar · cerrar venta · completar perfil)"*

### Q2 · Output que reinvierte
*"¿Esa acción genera output que ATRAE más usuarios? (testimonio · review · referido · contenido viral · case study)"*

### Q3 · Barreras
*"¿Qué frena que escale? (CAC alto · saturación canal · NPS bajo · capacidad)"*

### Q4 · Frecuencia
*"¿Con qué frecuencia se completa? (días · semanas · meses)"*

### Produce

```
LOOP PRINCIPAL: {nombre}
Frecuencia: ...

PASOS:
1. Usuario hace {core action}
2. Output: ...
3. Output atrae a {tipo de nuevo usuario}
4. Nuevo usuario hace {core action} → repite

INPUT QUE ESCALA: ...
GUARDRAIL: ...

¿ES REAL LOOP? ✅ / ❌
  Check: output > input → SÍ. Si no → es funnel.

SUB-LOOPS
```

### Itera

*"¿Hacemos el loop más corto? ¿Identifico canales que son puro funnel? ¿Te paso a /dashboard-builder?"*

## Reglas

- 1 loop principal · si dices 2 → no entiendes el modelo
- Test: output ≥ input · si no, NO es loop
- Frecuencia explícita
- Guardrail no negociable

## Ejemplo HC

**Input**: core action = intervención exitosa · output = testimonio antes/después · barrera = pacientes que no quieren foto · frecuencia 3 meses

**Output**:
- LOOP: Paciente operado → testimonio visual → lead orgánico → paciente operado
- ¿Real? ✅ — testimonios generan ~30% de leads
- INPUT escala: % pacientes que dan testimonio (hoy 25%, podría 50%)
- GUARDRAIL: calidad resultado · si baja, NPS cae, loop rompe
- SUB-LOOPS: programa referidos · partnership barberos
